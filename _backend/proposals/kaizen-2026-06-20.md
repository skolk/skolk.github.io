## Ratified direction acted on

The build-verdict friction recurs in **3 of the last 3** STATUS cycle entries (2026-06-17 08:06Z,
08:51Z, 17:20Z) plus ADR-003. Defining evidence: the sandboxed diagnostician failed in *both*

over-report (17:20Z, "10 broken images" that reconciled to 0). Root cause: a verdict derived from a
tool the agent could not run is untrustworthy in either sign. The fix targets verdict *provenance*,
not polarity.

## Proposed gated edits (each paired with a guarding fixture)

1. **Main-thread build as a hard, gating step** (`commands/run-project-cycle.md` + byte-identical
   twin `project-cycle.md`, R-005). Sub-agent "looks clean" is advisory only; build-state is sourced
   from the main thread's actual exit code.
   - Fixture `build-verdict-provenance/`: seed a failing build + an agent transcript claiming "no
     static blockers"; PASS proves synthesis won't promote the false-green.

2. **Forbid sub-agents from emitting tool-derived verdicts they didn't run** (diagnostician agent
   spec). Every build/lint/grep finding must be tagged `[ran: cmd, exit N]` or `[static-read]`;
   `[static-read]` is a hypothesis, never settled.
   - Fixture `diagnostician-static-read-tagging/`: seeds the exact 17:20Z trap (commented-out img
     refs + unused partials); PASS closes both over- and under-report directions, including the
     em-dash grep case Edit 1 doesn't cover.

3. **Synthesis must reconcile `[static-read]` counts against main-thread reality** (both command
   files). A bare unreconciled count can't enter the run summary. Reuses Edit 2's fixture with a
   synthesis-stage assertion.

4. **Build-state recorded as a typed enum** `GREEN | BROKEN | UNVERIFIED-STALL | UNVERIFIED` (both
   command files), so exit-0 / exit-124-stall / not-run can never blur into "green."
   - Fixture `build-state-enum/`: three seeded exits (0/1/124) assert the matching enum value.

**Recommended bump:** minor (v0.6.0 → v0.7.0) — additive hard-step + new contracts that change what
a PASS may assert.
